## MediaFlow 媒体整理方案

> 定义"目录组织 + 整理执行 + 信息刮削 + 分类实现"的完整策略。

---

## 1. 目录结构设计

### 1.1 设计原则

- **稳定性**：同一媒体在不同时间/来源整理结果一致
- **可扩展**：支持新增类型而不影响既有规则
- **原始保护**：不修改原始下载目录，通过硬链/软链/复制/移动方式整理
- **媒体服务器兼容**：符合 Plex/Jellyfin/Emby 扫描规范

### 1.2 目录层级规则

#### 1.2.1 电影目录

**结构**：`Movies/{类型}/{Title} ({Year})/` 或 `Movies/{类型}/{CollectionName}/{Title} ({Year})/`

- **一级目录**：`Movies/`
- **二级目录**（可配置）：
  - `电影/`：真人电影（默认）
  - `动画电影/`：动画长片（TMDB Genre 包含 Animation）
- **三级目录**：
  - 非合集：`{Title} ({Year})/`
  - 合集（可选）：`{CollectionName}/{Title} ({Year})/`（需外部元数据明确给出 collection）
- **规则**：
  - 同名不同年严格分开
  - 同一电影多版本在同目录内共存，用版本字段区分

**示例**：
```
Movies/电影/肖申克的救赎 (1994)/肖申克的救赎.1994.1080p.BluRay.x265.mkv
Movies/动画电影/千与千寻 (2001)/千与千寻.2001.2160p.BluRay.x265.mkv
Movies/电影/指环王系列/指环王1：护戒使者 (2001)/...
```

#### 1.2.2 剧集目录

**结构**：`TV/{类型}/{ShowTitle}/Season {Season}/`

- **一级目录**：`TV/`
- **二级目录**（按内容类型，可配置）：
  - `动漫/`、`电视剧/`、`纪录片/`、`综艺/`、`儿童/`、`短剧/`、`体育/`、`新闻/`、`教育/`、`访谈/`、`其他/`
- **三级目录**：`{ShowTitle}/`（剧集系列名称）
- **四级目录**：`Season 01/`、`Season 02/`（归一化为两位数字）

**类型识别映射**（优先级从高到低）：
```
TMDB Genre          → 二级目录
Animation           → 动漫/
Documentary         → 纪录片/
Kids/Family         → 儿童/
Reality/Talk Show   → 综艺/或访谈/
News                → 新闻/
Drama/Crime/...     → 电视剧/
集数<20 且单集<20分钟 → 短剧/（技术字段判断）
无法匹配            → 其他/
```

**示例**：
```
TV/动漫/进击的巨人/Season 01/进击的巨人.S01E01.1080p.WEB-DL.x264.mkv
TV/电视剧/Breaking Bad/Season 03/Breaking.Bad.S03E05.720p.BluRay.x264.mkv
TV/纪录片/地球脉动/Season 02/地球脉动.S02E01.2160p.WEB-DL.x265.mkv
```

**特殊情况**：
- 单季剧也保持 `Season 01/` 结构
- 无季概念的综艺可用 `Season 2024/` 或统一 `Season 01/`

#### 1.2.3 兜底目录

**结构**：`Unclassified/{ParentName}/`

- 用于无法刮削的内容（课程/会议/监控等）
- ParentName 取父目录归一化名称
- 命名：`{ParentName}.#{Index:03}.{Resolution}.{Codec}.mkv`

### 1.3 文件命名规则

**模板**（可配置）：
- 电影：`{Title}.{Year}.{Resolution}.{Source}.{Codec}-{Group}`
- 剧集：`{ShowTitle}.S{Season:02}E{Episode:02}.{Resolution}.{Source}-{Group}`

**字段说明**：
- **身份字段**：Title / ShowTitle / Year / Season / Episode
- **版本字段**：Resolution / Source / Codec / Audio / EditionTag / LanguageTag / Group

**约束**：
- 非法字符替换为 `_` 或移除（Windows/NAS 兼容）
- 路径超长时截断次要字段，完整信息保留在数据库

## 2. 整理执行策略

### 2.1 整理方式（可配置）

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **硬链接**（推荐） | 不占额外空间，删除任一端不影响另一端 | 需同盘同文件系统 | 下载与媒体库同盘 |
| **软链接** | 可跨盘，不占空间 | 媒体服务器支持度不一，删除原文件会失效 | 跨盘整理 |
| **复制** | 完全独立，删除原文件不受影响 | 占用双倍空间 | 需独立备份或跨盘不支持硬链 |
| **移动** | 不占额外空间，文件彻底迁移 | 删除原始文件 | 不需保留原始文件或空间紧张 |

**原则**：
- 整理操作不修改/删除原始下载目录（移动模式除外）
- 整理后内容统一放置在独立目标目录（如 `/media/library/`）

### 2.2 冲突处理（可配置）

**场景**：目标路径已存在同名文件

**策略**：
1. **加后缀**（推荐）：`(1)`/`(2)` 或 `.{hash8}`，适用所有方式
2. **智能覆盖**：校验为同一文件（size+hash 或 inode）时覆盖，否则加后缀
3. **跳过**：记录为 `skipped`，不创建目标文件

**特殊处理**：
- 硬链/软链：冲突时优先检测 inode 是否一致（已链接过），一致则跳过
- 移动：建议优先跳过或加后缀，避免误删原文件

### 2.3 分类视图（可选）

在主整理目录外，按 Genre/Year/Resolution 等维度生成 `Classified/...` 附加目录：
- **实现**：硬链/软链指向主整理目录，避免再次复制
- **用途**：提供多维度浏览或兼容特定外部工具
- **开关**：默认不生成，按需启用

## 3. 整理流程

### 3.1 流程步骤

```
1. 解析：提取标题/年份/季集/版本标签
2. 分组：形成合集，标记多版本
3. 刮削：补全身份字段（按需调用外部元数据）
4. 命名：按模板生成目标目录/文件名
5. 冲突检测：应用冲突处理策略
6. 生成计划：产出源→目标映射（默认 Dry-run）
7. 执行整理：
   - 按用户配置执行硬链/软链/复制/移动
   - 可分批执行，逐条返回结果（成功/失败/跳过）
   - 记录审计日志（源路径/目标路径/方式/时间戳）
8. 落库：写入媒体条目与分类字段
9. （可选）生成分类视图
```

### 3.2 识别规则

- **电影**：以"标题+年份"为主键，同名不同年不合并
- **剧集**：按"剧名+季"分组，同季按集号，同集多版本归组
- **多版本区分**：分辨率、来源、编码、版本标签、音轨/字幕
- **归一化**：去标点、统一大小写、去冗余空格

### 3.3 置信度分级

| 级别 | 匹配条件 | 处理策略 |
|------|---------|---------|
| **P1 精确** | 外部 ID 明确 或 Title+Year/Season/Episode 全匹配 | 允许自动执行 |
| **P2 可接受** | 标题归一化+年份一致，少量字段缺失 | 可执行但标记 warning |
| **P3 低置信度** | 相似度达标但存在歧义（同名/多候选） | 仅 Dry-run 待确认 |
| **P4 启发式** | 仅靠父目录/关键词猜测 | 一律不自动执行 |

## 4. 媒体信息刮削

### 4.1 刮削目标

- **身份字段**：Title+Year / ShowTitle+Season+Episode + 外部 ID（TMDB/IMDB/TVDB）
- **展示字段**：简介、海报、类型、地区、语言、上映日期、评分、状态
- **技术字段**：分辨率、编码、时长、音轨/字幕语言、HDR/DV

### 4.2 数据来源优先级

1. **文件名/目录名解析**：零成本，可能噪声大
2. **本地已刮削（NFO/媒体库缓存）**：结构化且稳定，优先复用
3. **媒体探针（ffprobe/mediainfo）**：补齐技术字段，并发/IO 可控
4. **外部元数据（TMDB 等）**：正规化身份与补全展示字段，需缓存/限速

### 4.3 触发时机

- **计划生成前**：先解析+初步分组
- **仅在需要时请求外部**：
  - 缺关键身份字段（Year/Season/Episode）
  - 低置信度匹配（相似度不足/同名歧义）
  - 用户显式要求"强制刷新"
- **执行后落库**：写入最终媒体条目与外部 ID

### 4.4 缓存与限速

- **缓存键**：外部 ID 优先；否则使用 `normalized_title+year` / `show+season`
- **缓存策略**：成功长缓存（7~30天）、失败短缓存（10~60分钟）
- **限速**：按 provider 做 QPS/并发控制，失败指数退避
- **审计**：记录请求/命中缓存/候选列表/最终选择与置信度

### 4.5 无法刮削内容兜底

- **场景**：课程/会议/监控等无外部元数据
- **流程**：序号/时间提取 → 排序命名 → （可选）探针补足技术字段 → 归入 `Unclassified/{ParentName}/`
- **命名**：`{ParentName}.#{Index:03}.{Resolution}.{Codec}.mkv`
- **标记**：结果标记为"未刮削/基础整理"

## 5. 分类策略与实现

### 5.1 分类形态

- **落库字段（主形态）**：对每个 MediaItem 写入可检索字段与多值关系（Genres/Tags）
- **可选分类视图**：`Classified/...` 目录树，用链指向主整理目录

### 5.2 分类维度

- **类型**：Action/Drama/Comedy/Sci-Fi/Horror/Animation/Documentary
- **地区/语言**：CN/US/JP/KR/UK/FR/ES + LanguageTag（CHS/CHT/ENG/JPN/Multi）
- **年份/年代**：单一年份（2024）或年代分箱（2020s/2010s/2000s）
- **分辨率/介质**：2160p/1080p/720p + BluRay/WEB-DL/WEBRip/HDTV/Remux
- **评分与热度**：TMDB/IMDB 评分 + 下载/播放热度
- **版本标记**：Extended/Director's Cut/IMAX/Remux
- **标签体系**：用户自定义标签（儿童/家庭/节日/学习）
- **状态**：已完成/在播/未完结，是否缺集

### 5.3 数据来源

1. **外部元数据**：类型、地区、语言、年份、评分、状态（需缓存/限速）
2. **文件/探针**：分辨率、介质、编码、音轨语言、时长
3. **本地标注**：用户手工或批量规则赋值
4. **业务派生**：热度（播放/下载次数）、缺集状态

### 5.4 多值策略

- **多类型/多地区**：列表存储，首选项用于主分类目录，其余写入标签/多值字段
- **未知/缺失**：统一归为 `Unknown`（数据库 NULL 或枚举），UI 可筛"待补全"
- **派生字段**：年代段/缺集状态/热度等级由后台定期计算或入库时计算

### 5.5 分类目录生成（可选）

**示例**：
- `Classified/Genre/{Genre}/{Title (Year)}/`
- `Classified/Region/{Region}/{Title (Year)}/`
- `Classified/Year/{Year}/{Title}/`
- `Classified/Resolution/{Resolution}/{Title}/`

**实现**：
- 默认不生成（仅落库），按需异步批处理
- 用硬链/软链指向主整理目录，避免复制
- 冲突使用 `Title (Year) [tmdb-<id>]` 确保唯一

## 附录：术语说明

### 命名字段

- **Title**：电影片名（如 `The Shawshank Redemption`、`肖申克的救赎`）
- **ShowTitle**：剧集系列名称（如 `Breaking Bad`、`三体`），非单集标题
- **OriginalTitle**：原始语言标题
- **Year**：上映/首播年份
- **Season**：季号（01/02），用于剧集组织
- **Episode**：集号（01/12），单季内集数编号

### 版本/技术字段

- **Resolution**：分辨率（2160p/1080p/720p/480p）
- **Source**：来源/介质（BluRay/WEB-DL/WEBRip/HDTV/DVDRip/Remux）
- **Codec**：视频编码（x264/x265/AV1/HEVC）
- **Audio**：音频编码（AAC/DDP5.1/DTS-HD.MA/TrueHD.Atmos）
- **EditionTag**：版本标记（Extended/Director's Cut/IMAX/Theatrical）
- **LanguageTag**：语言/字幕标记（CHS/CHT/ENG/JPN/Multi）
- **Group**：发布组/制作组名称（可为空）

### 整理方式

- **硬链接**：多个文件名指向同一 inode，删除任一不影响其他，不占额外空间，需同盘同文件系统
- **软链接**：类似快捷方式，指向原文件路径，可跨盘，删除原文件会失效
- **复制**：完整复制文件内容，两份文件独立，占用双倍空间
- **移动**：文件迁移到目标位置，原位置删除，不占额外空间（同盘是重命名，跨盘是复制+删除）

### 组织概念

- **原始目录**：下载器下载完成后的原始文件位置，整理不修改/删除（移动模式除外）
- **主整理目录**：整理后统一管理的目标目录（Movies/TV），通过硬链/软链/复制/移动生成
- **分类视图**：基于分类维度生成的附加目录树（Classified/...），用链指向主整理目录
- **Collection**：合集/系列（如指环王系列、漫威电影宇宙）
- **Genre**：类型/体裁（Action/Drama/Comedy/Sci-Fi/Horror）

### 流程术语

- **Dry-run**：模拟执行模式，生成整理计划但不实际执行，用于预览与确认
- **归一化**：将标题/文件名转换为标准格式（去标点/统一大小写/去冗余空格），便于匹配
- **置信度**：匹配结果可信程度，分为 P1（精确）、P2（可接受）、P3（低置信度）、P4（启发式）
- **多版本**：同一媒体的不同发行版本（不同分辨率/剪辑版/音轨等）
- **ParentName**：父目录名称，用于无法刮削内容的兜底整理

### 剧集内容类型（TV 二级目录）

- **动漫**：动画/番剧（TMDB Animation）
- **电视剧**：真人连续剧（Drama/Crime/Mystery/Thriller）
- **纪录片**：纪录片/记录片（TMDB Documentary）
- **综艺**：综艺节目/真人秀（TMDB Reality/Talk Show）
- **儿童**：儿童节目/学龄前动画（TMDB Kids/Family）
- **短剧**：短剧/网剧/微短剧（集数<20 且单集<20分钟）
- **体育**：体育节目/赛事直播
- **新闻**：新闻节目（TMDB News）
- **教育**：教育/学习/课程类
- **访谈**：访谈节目/脱口秀（TMDB Talk）
- **其他**：无法明确归类或混合类型
